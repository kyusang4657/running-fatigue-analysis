# Running Fatigue Analysis

## 프로젝트 개요

본 프로젝트는 **공개 러닝·웨어러블 데이터**와 **직접 수집한 Galaxy 스마트폰 IMU 데이터**를 활용하여 러닝 중 피로 변화 패턴을 분석하는 연구 프로젝트입니다.

연구 범위는 **러닝 피로 변화 분석**으로 한정합니다. AI Hub 또는 공개 데이터셋을 활용할 경우에도 러닝, 웨어러블 센서, IMU, 심박수, 러닝 로그와 관련된 데이터를 우선적으로 검토합니다. 보행 또는 일반 운동 데이터는 러닝 데이터가 부족할 경우 feature 설계 참고용으로만 활용합니다.

직접 수집 데이터는 Galaxy 스마트폰의 IMU 센서 데이터를 핵심으로 사용하며, Galaxy Fit3와 Samsung Health에서 확인할 수 있는 심박수, 수면 시간, 운동 기록 등은 선택적인 보조 데이터로 활용합니다.

본 프로젝트는 삼성 헬스, Garmin, Strava와 같은 상용 러닝 앱을 대체하는 것이 아니라, 상용 앱의 거리·페이스 중심 기록을 보완하는 **설명 가능한 러닝 피로 변화 분석 시스템**을 목표로 합니다.

---

## 연구 주제

**공개 러닝·웨어러블 데이터 기반 모델과 개인 기준선 RFCI를 활용한 러닝 피로 변화 분석 연구**

---

## 연구 배경

상용 러닝 앱은 거리, 시간, 페이스, 심박수, 칼로리 등 다양한 운동 기록을 제공한다. 그러나 사용자가 러닝 후반부에 피로를 느꼈을 때, 그 피로가 어떤 센서 변화에서 비롯되었는지 설명하는 데에는 한계가 있다.

기존 연구에서는 IMU 센서나 웨어러블 센서를 이용한 운동 상태 분석, 피로도 분류, 자세 분석 등이 수행되어 왔다. 하지만 공개 데이터 기반 모델이 실제 개인 러닝 데이터에 얼마나 잘 적용되는지, 그리고 개인별 초반 러닝 패턴을 기준선으로 삼아 후반부 피로 변화를 설명할 수 있는지는 별도로 검토할 필요가 있다.

---

## 핵심 아이디어

본 프로젝트는 다음 두 가지 접근을 결합한다.

1. **공개 러닝·웨어러블 데이터 기반 baseline**
   - AI Hub 또는 공개 데이터셋에서 러닝·웨어러블·IMU·심박수 관련 데이터를 조사한다.
   - 데이터 구조와 라벨을 검토하고, 가능한 경우 baseline 모델 또는 feature 설계에 활용한다.

2. **직접 수집 데이터 기반 개인 기준선 RFCI**
   - Galaxy 스마트폰 IMU 데이터를 직접 수집한다.
   - 러닝 초반 구간을 개인 기준선으로 설정한다.
   - 후반 구간의 페이스 저하율, 상하 흔들림 증가율, 좌우 흔들림 증가율, 리듬 불안정성 증가율을 계산한다.
   - 이러한 변화량을 바탕으로 RFCI, 즉 Running Fatigue Change Index를 계산한다.

## 기존 상용 앱과의 차이점

| 구분 | 상용 러닝 앱 | 본 프로젝트 |
|---|---|---|
| 목적 | 운동 기록 관리, 코칭 제공 | 피로 변화 분석 방식의 실험과 검증 |
| 핵심 데이터 | 거리, 시간, 페이스, 심박수 | 공개 러닝 데이터, 스마트폰 IMU, 개인 기준선 |
| 분석 방식 | 상용 알고리즘 기반 | 공개 데이터 baseline + 개인 기준선 변화량 분석 |
| 결과 표현 | 운동 기록 요약, 훈련 계획 | 피로 변화 지수와 판단 근거 제공 |
| 목표 | 사용자 서비스 | 연구 및 실험 검증 |

- 공개 데이터 기반 분석 결과와 개인 기준선 RFCI 결과 비교
- Streamlit 기반 분석 대시보드 구현

## 연구 범위와 주의사항

본 프로젝트의 핵심 범위는 **러닝 피로 변화 분석**이다. 일반 운동 전체를 분석하지 않는다.

본 프로젝트는 의학적 진단, 부상 예측, 치료 조언을 목적으로 하지 않는다. 분석 결과는 운동 데이터 기반 피로 변화 패턴을 이해하기 위한 보조 정보로만 사용한다.

AI Hub 또는 공개 데이터셋의 원본 데이터는 라이선스와 이용 조건을 확인한 뒤 사용하며, 원본 대용량 데이터는 GitHub에 업로드하지 않는다.

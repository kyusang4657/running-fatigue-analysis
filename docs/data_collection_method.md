# 데이터 수집 방법

## 1. 목적

본 문서는 직접 수집하는 러닝 데이터의 수집 방법을 정리하기 위한 문서이다.

본 연구는 Galaxy 스마트폰의 IMU 센서 데이터를 핵심 데이터로 사용하고, Galaxy Fit3와 Samsung Health에서 확인할 수 있는 운동 기록, 심박수, 수면 시간 등은 선택 보조 데이터로 활용한다.

AI Hub 또는 공개 데이터셋은 별도 문서인 `docs/aihub_dataset_review.md`에서 조사하며, 본 문서에서는 Galaxy 스마트폰, Galaxy Fit3, Samsung Health를 활용한 **직접 러닝 데이터 수집 방법**만 다룬다.

---

## 2. 현재 테스트 환경

| 구분 | 내용 |
|---|---|
| 스마트폰 | Galaxy 스마트폰 |
| 웨어러블 | Galaxy Fit3 |
| 건강 앱 | Samsung Health |
| 개발 환경 | Python, PyCharm |
| 데이터 형식 | CSV 중심 |

## 3. 데이터 수집 구조

본 연구의 직접 데이터 수집 구조는 다음과 같다.

```text
Galaxy 스마트폰
  └─ IMU 센서 데이터 수집
      ├─ accelerometer: acc_x, acc_y, acc_z
      └─ gyroscope: gyro_x, gyro_y, gyro_z

Galaxy Fit3 + Samsung Health
  └─ 선택 보조 데이터 확인
      ├─ 평균 심박수
      ├─ 최대 심박수
      ├─ 수면 시간
      ├─ 최근 운동 기록
      └─ 운동 거리 / 시간 / 페이스
```

직접 수집 데이터는 대규모 모델 학습용이 아니라, 개인 기준선 RFCI 계산 가능성과 공개 데이터 기반 분석 방법의 외부 적용 가능성을 확인하기 위한 파일럿 테스트 데이터로 사용한다.

---

## 4. 스마트폰 IMU 데이터 수집 방법

### 4.1 수집 대상 센서

| 센서 | 컬럼 | 설명 |
|---|---|---|
| Accelerometer | acc_x, acc_y, acc_z | x, y, z축 가속도 |
| Gyroscope | gyro_x, gyro_y, gyro_z | x, y, z축 회전 변화 |
| Time | timestamp | 센서 측정 시간 |

초기 실험에서는 가속도계 데이터를 필수로 수집하고, 자이로스코프 데이터는 가능하면 함께 수집한다.

### 4.2 수집 앱 후보

초기 테스트에서는 스마트폰 센서 기록 앱을 사용한다. 앱 선택 시 다음 항목을 확인한다.

- 가속도계 x, y, z 데이터 저장 가능 여부
- 자이로스코프 x, y, z 데이터 저장 가능 여부
- timestamp 포함 여부
- CSV export 가능 여부
- 러닝 중 백그라운드 기록 가능 여부
- 화면 꺼짐 상태에서도 기록 유지 가능 여부

---

## 5. Samsung Health 보조 데이터 확인 방법

Samsung Health와 Galaxy Fit3는 본 연구의 선택 보조 데이터 수집에 활용한다.

| 데이터 | 설명 | 필수 여부 |
|---|---|---|
| 평균 심박수 | 러닝 중 평균 심박수 | 선택 |
| 최대 심박수 | 러닝 중 최대 심박수 | 선택 |
| 수면 시간 | 전날 수면 시간 | 선택 |
| 최근 7일 운동량 | 최근 7일 러닝 거리 또는 운동 시간 | 선택 |
| 러닝 거리 | Samsung Health에 기록된 거리 | 선택 |
| 러닝 시간 | Samsung Health에 기록된 시간 | 선택 |
| 평균 페이스 | Samsung Health에 기록된 평균 페이스 | 선택 |

초기 연구에서는 자동 연동보다 수동 기록 또는 다운로드 파일 확인을 우선한다.

## 6. 데이터 저장 위치

수집한 원본 데이터는 다음 위치에 저장한다.

```text
data/raw/
```

예시:

```text
data/raw/runnerA_run001_sensor.csv
data/raw/runnerA_run001_log.csv
```

단, `data/raw/`는 `.gitignore`에 의해 GitHub에 업로드하지 않는다. 공개 가능한 예시 데이터만 `data/sample/`에 저장한다.

---

## 7. 파일 이름 규칙

센서 데이터와 러닝 로그는 다음 규칙으로 저장한다.

```text
runnerID_run번호_sensor.csv
runnerID_run번호_log.csv
```

예시:

```text
runnerA_run001_sensor.csv
runnerA_run001_log.csv
```

## 8. 초기 테스트 절차

초기 테스트는 실제 러닝 전에 짧은 걷기 또는 제자리 뛰기로 수행한다.

### 테스트 1. 스마트폰 센서 기록 확인

1. Galaxy 스마트폰에 센서 기록 앱을 설치한다.
2. 가속도계와 자이로스코프 기록을 시작한다.
3. 1분간 걷기 또는 제자리 뛰기를 수행한다.
4. 기록을 중지한다.
5. CSV로 내보낸다.
6. CSV 파일에 timestamp, acc_x, acc_y, acc_z, gyro_x, gyro_y, gyro_z가 포함되어 있는지 확인한다.

### 테스트 2. Samsung Health 보조 데이터 확인

1. Galaxy Fit3를 착용한다.
2. Samsung Health에서 걷기 또는 러닝 운동을 시작한다.
3. 5~10분 정도 가볍게 움직인다.
4. 운동 종료 후 Samsung Health에서 운동 기록을 확인한다.
5. 평균 심박수, 최대 심박수, 운동 시간, 거리, 페이스가 확인 가능한지 기록한다.

---

## 9. 실제 러닝 수집 절차

실제 러닝 데이터는 다음 절차로 수집한다.

1. 스마트폰을 허리 또는 러닝 벨트에 고정한다.
2. 센서 기록 앱에서 IMU 데이터 기록을 시작한다.
3. Samsung Health에서 러닝 운동 기록을 시작한다.
4. 10~20분 러닝을 수행한다.
5. 운동 종료 후 센서 기록과 Samsung Health 기록을 저장한다.
6. 러닝 직후 주관적 피로도 1~5점을 기록한다.
7. 원본 데이터는 `data/raw/`에 저장한다.
8. 공개 가능한 예시 데이터만 익명화하여 `data/sample/`에 저장한다.

## 10. 성공 기준

Issue #2는 다음 조건을 만족하면 완료한다.

- 스마트폰 IMU 데이터를 CSV로 저장할 수 있다.
- CSV 파일에 timestamp와 가속도계 x, y, z 데이터가 포함되어 있다.
- 가능하면 자이로스코프 x, y, z 데이터도 포함되어 있다.
- Samsung Health에서 평균 심박수, 운동 시간, 거리, 페이스 중 어떤 항목을 확인할 수 있는지 정리했다.
- 샘플 데이터 파일을 `data/sample/`에 저장했다.
- 개인정보가 포함될 수 있는 원본 데이터는 GitHub에 업로드하지 않았다.

---

## 11. 개인정보 및 안전 주의사항

- 실제 이름 대신 runner_A, runner_B와 같은 익명 ID를 사용한다.
- GPS 좌표나 정확한 위치 정보는 공개 저장소에 업로드하지 않는다.
- 원본 운동 데이터는 `data/raw/`에 저장하고 GitHub에는 올리지 않는다.
- 공개 가능한 샘플 데이터만 `data/sample/`에 저장한다.
- 본 연구는 의학적 진단이 아니라 운동 데이터 기반 피로 변화 분석을 목적으로 한다.

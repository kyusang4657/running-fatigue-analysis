# 데이터 수집 양식

## 1. 목적

본 문서는 러닝 피로 변화 분석 연구에서 수집할 데이터 항목을 정의하기 위한 문서이다.

본 연구는 스마트폰 IMU 센서와 러닝 로그를 활용하여 러닝 초반 구간 대비 후반 구간의 움직임 변화와 피로 변화를 분석하는 것을 목표로 한다.

스마트폰 IMU 데이터는 필수 데이터로 사용하며, Galaxy Fit3와 Samsung Health에서 확인할 수 있는 심박수, 수면 시간, 최근 운동량 등은 선택 보조 데이터로 활용한다. 단, 팀원별 보유 기기 차이를 고려하여 웨어러블 데이터는 필수 조건이 아닌 보조 feature로 처리한다.

---

## 2. 러닝 1회당 기본 기록 항목

| 항목                 | 설명          | 예시                       | 필수 여부 |
| ------------------ | ----------- | ------------------------ | ----- |
| run_id             | 러닝 기록 고유 ID | run_001                  | 필수    |
| date               | 러닝 날짜       | 2026-07-01               | 필수    |
| runner_id          | 참여자 ID      | runner_A                 | 필수    |
| distance_km        | 러닝 거리       | 5.0                      | 필수    |
| total_time_min     | 전체 러닝 시간    | 32.5                     | 필수    |
| avg_pace           | 평균 페이스      | 6:30/km                  | 필수    |
| phone_position     | 스마트폰 착용 위치  | waist                    | 필수    |
| route_type         | 러닝 장소 유형    | track / road / treadmill | 필수    |
| subjective_fatigue | 주관적 피로도     | 1~5                      | 필수    |
| memo               | 특이사항        | 더운 날씨, 수면 부족 등           | 선택    |

---

## 3. 스마트폰 IMU 센서 항목

스마트폰 센서 기록 앱 또는 별도 수집 도구를 통해 다음 항목을 CSV로 저장한다.

| 항목        | 설명        | 필수 여부   |
| --------- | --------- | ------- |
| timestamp | 센서 측정 시각  | 필수      |
| acc_x     | x축 가속도    | 필수      |
| acc_y     | y축 가속도    | 필수      |
| acc_z     | z축 가속도    | 필수      |
| gyro_x    | x축 자이로스코프 | 가능하면 필수 |
| gyro_y    | y축 자이로스코프 | 가능하면 필수 |
| gyro_z    | z축 자이로스코프 | 가능하면 필수 |

초기 연구에서는 가속도계 데이터를 필수로 수집하고, 자이로스코프 데이터는 수집 가능할 경우 함께 기록한다.

---

## 4. 선택 보조 데이터 항목

Galaxy Fit3와 Samsung Health 데이터를 사용할 수 있는 경우 다음 항목을 추가로 기록한다. 다른 팀원이 동일한 기기를 보유하지 않을 수 있으므로, 해당 항목은 필수 데이터가 아니라 선택 보조 데이터로 취급한다.

| 항목                      | 설명          | 예시                                        |
| ----------------------- | ----------- | ----------------------------------------- |
| device_type             | 사용 기기 종류    | smartphone only / smartphone + smartwatch |
| watch_model             | 스마트워치 모델    | Galaxy Fit3                              |
| avg_heart_rate          | 평균 심박수      | 155                                       |
| max_heart_rate          | 최대 심박수      | 178                                       |
| sleep_hours             | 전날 수면 시간    | 6.5                                       |
| recent_7day_distance_km | 최근 7일 러닝 거리 | 18.0                                      |
| cadence                 | 케이던스        | 165 spm                                   |
| recovery_note           | 컨디션 메모      | 전날 수면 부족                                  |

선택 보조 데이터는 모든 참여자가 반드시 수집해야 하는 항목은 아니다.
Galaxy Fit3와 Samsung Health 데이터를 확보할 수 있는 경우에만 추가로 기록한다.

## 5. 주관적 피로도 기준

러닝 직후 참여자가 직접 피로도를 1~5점으로 기록한다.

| 점수 | 의미         |
| -- | ---------- |
| 1  | 거의 피곤하지 않음 |
| 2  | 약간 피곤함     |
| 3  | 보통 수준의 피로  |
| 4  | 많이 피곤함     |
| 5  | 매우 피곤함     |

주의: 본 피로도는 의학적 진단이 아니라 연구 참여자가 느낀 주관적 운동 피로 수준을 의미한다.

---

## 6. 스마트폰 착용 위치

초기 연구에서는 스마트폰 위치를 고정하여 실험한다.

우선 권장 위치는 다음과 같다.

```text
waist
```

즉, 허리 벨트 또는 러닝 벨트에 스마트폰을 고정한다.

허리 위치를 우선 고려하는 이유는 다음과 같다.

* 몸 전체의 상하 흔들림을 비교적 잘 반영할 수 있음
* 팔 흔들림의 영향을 줄일 수 있음
* 러닝 중 위치 변화가 상대적으로 적음

---

## 7. 데이터 구간 분할 기준

러닝 데이터를 거리 또는 시간 기준으로 세 구간으로 나눈다.

| 구간    | 범위      | 역할          |
| ----- | ------- | ----------- |
| 초반 구간 | 0~20%   | 개인 기준선      |
| 중반 구간 | 20~70%  | 일반 러닝 구간    |
| 후반 구간 | 70~100% | 피로 변화 분석 구간 |

예를 들어 5km 러닝의 경우 다음과 같이 나눌 수 있다.

| 구간 | 거리      |
| -- | ------- |
| 초반 | 0~1km   |
| 중반 | 1~3.5km |
| 후반 | 3.5~5km |

## 8. 파일 이름 규칙

러닝 데이터 파일은 다음 규칙으로 저장한다.

```text
runnerID_run번호_sensor.csv
runnerID_run번호_log.csv
```

예시:

```text
runnerA_run001_sensor.csv
runnerA_run001_log.csv
```

## 10. 개인정보 및 안전 주의사항

* 실제 이름 대신 `runner_A`, `runner_B`와 같은 익명 ID를 사용한다.
* GPS 좌표나 정확한 위치 정보는 저장하지 않는다.
* 무리한 러닝을 강요하지 않는다.
* 부상, 질병, 건강 진단과 관련된 표현은 사용하지 않는다.
* 본 연구는 운동 데이터 기반 피로 변화 분석을 목적으로 한다.

# 초기 연구계획서

## 연구 주제

**개인 기준선 기반 스마트폰 IMU 러닝 피로 변화 지수 설계 및 설명 가능한 피드백 시스템**

---

## 1. 연구 주제 한 줄 요약

본 연구는 스마트폰 IMU 센서와 러닝 기록 데이터를 활용하여 러닝 중 피로 변화가 어떻게 나타나는지 분석하고, 개인별 초기 러닝 패턴을 기준으로 피로 변화 지수를 계산한 뒤, 사용자가 이해할 수 있는 피드백을 제공하는 시스템을 제안한다.

쉽게 말하면, 단순히 “몇 km를 뛰었다”, “페이스가 얼마다”를 보여주는 것이 아니라, **초반의 내 움직임과 후반의 내 움직임이 얼마나 달라졌는지**를 비교하여 피로 변화를 설명하는 연구이다.

---

## 2. 연구 배경

최근 러닝, 걷기, 수영 등 개인 운동 데이터를 기록하고 분석하는 서비스가 많이 사용되고 있다. 대표적으로 삼성 헬스, Garmin, Strava와 같은 앱은 거리, 시간, 페이스, 심박수, 칼로리, 운동 기록 등을 제공하며, 일부 기기는 러닝 코칭 기능도 제공한다.

하지만 이러한 상용 앱은 주로 사용자의 운동 결과를 요약하거나 훈련 계획을 제공하는 데 초점을 둔다. 반면 사용자가 운동 중 피로해졌을 때, **어떤 센서 변화 때문에 피로하다고 판단할 수 있는지**, 그리고 **개인의 평소 움직임과 비교했을 때 얼마나 변화했는지**를 구체적으로 설명하는 데에는 한계가 있다.

따라서 본 연구는 상용 러닝 앱을 대체하는 것이 아니라, 상용 앱의 운동 기록 기능을 보완하는 연구용 분석 시스템을 목표로 한다.

---

## 3. 문제 정의

기존 러닝 앱은 다음과 같은 정보를 잘 제공한다.

| 제공 정보 | 예시 |
|---|---|
| 거리 | 5km |
| 시간 | 32분 |
| 평균 페이스 | 6분 24초/km |
| 심박수 | 평균 155bpm |
| 칼로리 | 350kcal |
| 운동 기록 | 날짜별 러닝 기록 |

그러나 본 연구에서 주목하는 문제는 다음과 같다.

> 사용자가 러닝 후반부에 피로해졌다고 느꼈을 때, 그 피로가 센서 데이터상에서 어떤 변화로 나타나는가?

예를 들어 다음과 같은 질문에 답하고자 한다.

- 후반부에 상하 흔들림이 증가했는가?
- 좌우 흔들림이 커졌는가?
- 페이스가 초반보다 얼마나 저하되었는가?
- 움직임 리듬이 불안정해졌는가?
- 이러한 변화가 개인의 초반 기준선과 비교했을 때 의미 있는 변화인가?

---

## 4. 삼성 헬스와의 차이점

삼성 헬스와 같은 상용 앱이 이미 존재하기 때문에, 본 연구는 단순 운동 기록 앱을 만드는 것이 아니다. 차이점은 다음과 같다.

| 구분 | 삼성 헬스 등 상용 앱 | 본 연구 |
|---|---|---|
| 목적 | 운동 기록 관리, 건강 관리, 코칭 제공 | 피로 변화 분석 방식의 실험과 검증 |
| 핵심 데이터 | 거리, 시간, 페이스, 심박수, 칼로리 등 | 스마트폰 IMU 센서, 러닝 로그, 개인 기준선 |
| 분석 방식 | 완성된 상용 알고리즘 중심 | 개인 기준선 대비 변화량 분석 |
| 결과 표현 | 기록 요약, 운동 점수, 훈련 계획 | 피로 변화 지수와 판단 근거 제공 |
| 차별점 | 사용자 서비스 완성도 | 연구 질문, 실험 비교, 설명 가능성 |
| 목표 | 운동 앱으로 사용 | 피로 변화 분석 모델 검증 |

핵심 차이는 다음 문장으로 정리할 수 있다.

> 삼성 헬스는 운동 기록과 코칭을 제공하는 상용 서비스이고, 본 연구는 러닝 중 피로 변화가 어떤 센서 변화에서 발생했는지 개인 기준선 기반으로 설명하는 연구 시스템이다.

즉, 본 연구는 삼성 헬스와 경쟁하는 앱을 만드는 것이 아니라, **상용 앱이 제공하는 거리·페이스 중심 기록을 보완하는 설명 가능한 피로 변화 분석 레이어**를 만드는 것이다.

## 5. 기존 연구와의 차별점

기존 연구에서는 IMU 센서나 웨어러블 센서를 이용해 러닝 피로도를 분류하는 연구가 이미 존재한다. 따라서 “IMU 센서로 피로도를 분류한다”는 것만으로는 차별점이 약하다.

본 연구는 다음 세 가지 방향으로 차별점을 만든다.

### 5.1 개인 기준선 기반 변화량 분석

기존 연구는 여러 사람의 센서 데이터를 모아 하나의 피로 분류 모델을 만드는 경우가 많다. 하지만 사람마다 러닝 자세, 보폭, 흔들림, 페이스가 다르다.

따라서 본 연구는 러닝 초반 구간을 개인 기준선으로 설정하고, 이후 구간이 초반과 얼마나 달라졌는지를 분석한다.

예시:

| 항목 | 초반 구간 | 후반 구간 | 변화 |
|---|---:|---:|---:|
| 상하 흔들림 | 1.20 | 1.48 | +23.3% |
| 좌우 흔들림 | 0.85 | 1.02 | +20.0% |
| 페이스 | 6분 10초/km | 6분 55초/km | 저하 |
| 리듬 변동성 | 0.12 | 0.18 | +50.0% |

이 방식은 사람마다 원래 움직임이 다르다는 문제를 완화할 수 있다.

### 5.2 설명 가능한 피로 변화 지수

본 연구는 단순히 “피로도 높음”이라고 출력하지 않는다. 왜 피로도가 높게 나왔는지 판단 근거를 함께 제공한다.

예시:

```text
피로 변화 지수: 높음

판단 근거:
1. 후반 페이스가 초반 대비 12.1% 저하됨
2. 상하 흔들림이 18.4% 증가함
3. 좌우 흔들림이 9.8% 증가함
4. 리듬 불안정성이 15.6% 증가함
```

이러한 방식은 사용자가 결과를 더 쉽게 이해할 수 있게 한다.

### 5.3 훈련 피드백으로의 연결

기존 연구는 피로도 분류에서 끝나는 경우가 많다. 본 연구는 피로도 분석 결과를 바탕으로 다음 운동 강도에 대한 간단한 피드백을 제공한다.

예시:

| 피로 변화 상태 | 추천 피드백 |
|---|---|
| 피로 변화 낮음 | 평소 훈련 유지 |
| 피로 변화 중간 | 다음 운동은 가벼운 회복주 권장 |
| 피로 변화 높음 | 휴식 또는 짧은 저강도 러닝 권장 |
| 최근 훈련량 급증 | 과훈련 주의 |

단, 본 연구의 추천은 의학적 진단이 아니라 운동 데이터 기반 보조 피드백으로 한정한다.

---

## 6. 연구 목적

본 연구의 목적은 다음과 같다.

1. 스마트폰 IMU 센서와 러닝 로그를 이용해 러닝 중 피로 변화와 관련된 특징을 추출한다.
2. 러닝 초반 구간을 개인 기준선으로 설정하고, 이후 구간의 변화량을 분석한다.
3. 피로 변화 지수(RFCI, Running Fatigue Change Index)를 설계한다.
4. 피로 변화 판단 근거를 사용자가 이해할 수 있는 형태로 제공한다.
5. 기본적인 훈련 강도 피드백 시스템을 구현한다.

## 7. 연구 질문

본 연구의 핵심 연구 질문은 다음과 같다.

### 메인 연구 질문

> 개인별 초기 러닝 구간을 기준선으로 설정했을 때, 스마트폰 IMU 센서와 러닝 로그의 변화량을 통해 러닝 피로 변화를 설명 가능하게 분석할 수 있는가?

### 세부 연구 질문

1. 개인 기준선 대비 변화량 feature가 절대값 feature보다 피로 변화 분석에 더 적합한가?
2. 거리와 페이스 정보만 사용할 때보다 IMU 센서 변화량을 함께 사용할 때 피로 변화 추정이 더 안정적인가?
3. 피로 변화 지수를 사용할 경우 사용자가 피로 판단 근거를 더 쉽게 이해할 수 있는가?
4. 피로 변화 분석 결과를 간단한 훈련 강도 피드백으로 연결할 수 있는가?

## 8. 연구 가설

본 연구는 다음과 같은 가설을 가진다.

| 번호 | 가설 |
|---|---|
| H1 | 러닝 후반부에는 초반부에 비해 흔들림, 리듬 불안정성, 페이스 저하가 증가할 것이다. |
| H2 | 개인 기준선 대비 변화량 feature는 절대 센서값 feature보다 개인차를 줄이는 데 도움이 될 것이다. |
| H3 | 페이스 정보만 사용하는 것보다 IMU 센서 변화량을 함께 사용할 때 피로 변화 분석이 더 구체적일 것이다. |
| H4 | 피로 판단 근거를 함께 제공하면 단순 점수보다 사용자가 결과를 이해하기 쉬울 것이다. |

## 9. 데이터 수집 계획

### 9.1 수집 대상 데이터

| 데이터 종류 | 설명 |
|---|---|
| 가속도계 데이터 | x, y, z축 움직임 변화 |
| 자이로스코프 데이터 | x, y, z축 회전 변화 |
| 러닝 거리 | 총 거리 또는 구간별 거리 |
| 러닝 시간 | 전체 운동 시간 |
| 페이스 | km당 소요 시간 |
| 주관적 피로도 | 운동 후 1~5점으로 기록 |
| 운동 날짜 | 최근 훈련량 계산용 |

### 9.2 데이터 수집 방법

초기 연구에서는 복잡한 앱 개발보다 스마트폰 센서 기록 앱 또는 간단한 데이터 수집 도구를 활용한다.

가능한 방식:

1. 스마트폰 센서 기록 앱을 사용하여 IMU 데이터를 CSV로 저장한다.
2. 러닝 후 거리, 시간, 평균 페이스를 수동으로 기록한다.
3. 삼성 헬스와 같은 앱에서 확인한 러닝 기록을 참고 데이터로 활용할 수 있다.
4. 주관적 피로도는 러닝 직후 1~5점으로 입력한다.

### 9.3 스마트폰 착용 위치

스마트폰 위치가 바뀌면 센서 패턴도 달라지므로, 초기 실험에서는 위치를 고정한다.

추천 조건:

- 스마트폰을 허리 벨트 또는 러닝 벨트에 고정
- 또는 팔 밴드에 고정
- 실험 중 위치를 계속 바꾸지 않음

초기 연구에서는 **허리 고정**을 우선 고려한다. 허리 위치는 몸 전체의 흔들림을 비교적 잘 반영할 수 있기 때문이다.

## 10. 데이터 구간 분할

러닝 데이터를 다음과 같이 나눈다.

| 구간 | 의미 |
|---|---|
| 초반 0~20% | 개인 기준선 구간 |
| 중반 20~70% | 일반 러닝 구간 |
| 후반 70~100% | 피로 변화 분석 구간 |

예를 들어 5km를 달렸다면 다음과 같이 나눌 수 있다.

| 구간 | 거리 |
|---|---|
| 초반 | 0~1km |
| 중반 | 1~3.5km |
| 후반 | 3.5~5km |

이때 초반 구간의 센서 특징을 개인 기준선으로 사용하고, 후반 구간이 초반과 얼마나 달라졌는지 계산한다.

## 11. Feature 설계

본 연구에서 사용할 수 있는 주요 feature는 다음과 같다.

| Feature | 설명 |
|---|---|
| acc_mean | 가속도 평균 |
| acc_std | 가속도 표준편차 |

| lateral_sway_proxy | 좌우 흔들림 추정값 |

| fatigue_ratio | 후반 feature / 초반 feature |

초기 연구에서는 너무 많은 feature를 사용하기보다 다음 4개를 핵심으로 둔다.

1. 페이스 저하율
2. 상하 흔들림 증가율
3. 좌우 흔들림 증가율
4. 리듬 불안정성 증가율

## 12. 피로 변화 지수 RFCI 설계

본 연구에서는 피로 변화 지수를 임시로 RFCI라고 부른다.

**RFCI = Running Fatigue Change Index**

초기 버전의 RFCI는 다음 요소를 조합하여 계산한다.

| 구성 요소 | 의미 |
|---|---|
| 페이스 저하율 | 후반 페이스가 초반보다 얼마나 느려졌는가 |
| 상하 흔들림 증가율 | 후반부 상하 움직임이 얼마나 커졌는가 |
| 좌우 흔들림 증가율 | 후반부 좌우 움직임이 얼마나 커졌는가 |
| 리듬 불안정성 증가율 | 후반부 움직임 리듬이 얼마나 불규칙해졌는가 |

예시 계산 방식:

```text
RFCI = w1 * 페이스 저하율
     + w2 * 상하 흔들림 증가율
     + w3 * 좌우 흔들림 증가율
     + w4 * 리듬 불안정성 증가율
```

초기 연구에서는 모든 가중치를 동일하게 설정하고, 이후 실험 결과를 보며 조정할 수 있다.

```text
w1 = w2 = w3 = w4 = 0.25
```

RFCI 결과는 다음과 같이 해석한다.

| RFCI 범위 | 해석 |
|---|---|
| 낮음 | 피로 변화 적음 |
| 중간 | 피로 변화 관찰됨 |
| 높음 | 피로 변화 뚜렷함 |

정확한 기준값은 데이터 수집 후 실험적으로 정한다.

## 13. 모델 비교 실험 계획

본 연구는 단순히 하나의 모델만 만드는 것이 아니라, 여러 방식을 비교한다.

### 실험 1. 절대값 feature vs 개인 기준선 변화량 feature

| 비교 대상 | 설명 |
|---|---|
| 절대값 feature | acc_std, gyro_std 등의 원래 값 |
| 변화량 feature | 후반 feature / 초반 feature |

목적:

> 개인 기준선 대비 변화량 feature가 개인차를 줄이는 데 도움이 되는지 확인한다.

### 실험 2. 러닝 로그만 사용 vs IMU 추가 사용

| 비교 대상 | 사용 데이터 |
|---|---|
| 러닝 로그만 사용 | 거리, 시간, 페이스 |
| IMU만 사용 | 가속도, 자이로 |
| 러닝 로그 + IMU | 페이스 + 센서 변화량 |

목적:

> 상용 앱의 기본 기록 정보에 IMU 변화량을 추가했을 때 피로 분석이 더 구체적이 되는지 확인한다.

### 실험 3. 규칙 기반 지수 vs 머신러닝 모델

| 비교 대상 | 설명 |
|---|---|
| 규칙 기반 RFCI | 사람이 설계한 피로 변화 지수 |
| 머신러닝 모델 | Random Forest, SVM 등 |
| 비교 기준 | 정확도, F1-score, 해석 가능성 |

목적:

> 단순 규칙 기반 지수와 머신러닝 모델의 장단점을 비교한다.

## 21. 윤리 및 안전 고려사항

본 연구는 운동 데이터와 신체 활동 데이터를 다루기 때문에 다음 사항을 고려한다.

1. 수집한 데이터는 연구 목적 외에는 사용하지 않는다.
2. 개인 식별 정보는 저장하지 않는다.
3. 운동 중 무리한 실험을 강요하지 않는다.
4. 피로 분석 결과를 의학적 진단으로 표현하지 않는다.
5. 부상 위험, 질병 예측, 치료 조언과 같은 표현은 사용하지 않는다.

## 23. 최종 정리

본 연구는 기존 상용 러닝 앱을 대체하는 것이 아니라, 상용 앱에서 제공하는 거리와 페이스 중심의 운동 기록을 보완하는 연구이다.

핵심 차별점은 다음과 같다.

1. 개인별 초반 러닝 구간을 기준선으로 설정한다.
2. 후반부 움직임이 초반과 얼마나 달라졌는지 분석한다.
3. 피로 변화 지수 RFCI를 설계한다.
4. 피로 판단 근거를 사용자에게 설명한다.
5. 분석 결과를 간단한 훈련 피드백으로 연결한다.

최종적으로 본 연구는 다음 질문에 답하고자 한다.

> 스마트폰 IMU 센서와 개인 기준선 기반 변화량 분석을 통해 러닝 중 피로 변화를 설명 가능하게 분석할 수 있는가?

# 초기 연구계획서

## 연구 주제

**공개 러닝·웨어러블 데이터 기반 모델과 개인 기준선 RFCI를 활용한 러닝 피로 변화 분석 연구**

---

## 1. 연구 주제 한 줄 요약

본 연구는 AI Hub 또는 공개 러닝·웨어러블 데이터셋을 활용하여 러닝 피로 변화 분석에 필요한 데이터 구조와 feature를 검토하고, 직접 수집한 Galaxy 스마트폰 IMU 데이터와 Galaxy Fit3/Samsung Health 보조 데이터를 이용하여 개인 기준선 기반 러닝 피로 변화 지수 RFCI를 파일럿 검증하는 것을 목표로 한다.

쉽게 말하면, 공개 데이터로 기본적인 분석 기준을 잡고, 우리가 직접 수집한 러닝 데이터에서 **초반의 내 움직임과 후반의 내 움직임이 얼마나 달라졌는지**를 비교하여 피로 변화를 설명하는 연구이다.

---

## 2. 연구 범위

본 연구의 범위는 **러닝 피로 변화 분석**으로 한정한다.

AI Hub 또는 공개 데이터셋을 활용할 경우에도 러닝, 웨어러블, IMU 센서, 심박수, 러닝 로그와 관련된 데이터를 우선적으로 검토한다.

보행 데이터나 일반 운동 데이터는 러닝 데이터가 부족할 경우 feature 설계 참고용으로만 활용하며, 본 연구의 핵심 분석 대상에는 포함하지 않는다.

본 연구에서 제외하는 범위는 다음과 같다.

| 제외 범위 | 이유 |
|---|---|
| 일반 피트니스 자세 분석 | 러닝 피로 변화 분석과 목적이 다름 |
| 재활 운동 분석 | 의료·재활 목적이 강함 |
| 수영 동작 분석 | 센서 위치와 데이터 구조가 러닝과 다름 |
| 부상 진단 | 의학적 진단 영역에 해당할 수 있음 |
| 질병 예측 | 본 연구 범위를 벗어남 |

---

## 3. 연구 배경

최근 러닝, 걷기, 수영 등 개인 운동 데이터를 기록하고 분석하는 서비스가 많이 사용되고 있다. 대표적으로 삼성 헬스, Garmin, Strava와 같은 앱은 거리, 시간, 페이스, 심박수, 칼로리, 운동 기록 등을 제공하며, 일부 기기는 러닝 코칭 기능도 제공한다.

하지만 이러한 상용 앱은 주로 사용자의 운동 결과를 요약하거나 훈련 계획을 제공하는 데 초점을 둔다. 반면 사용자가 러닝 중 피로해졌을 때, **어떤 센서 변화 때문에 피로하다고 판단할 수 있는지**, 그리고 **개인의 초반 러닝 패턴과 비교했을 때 얼마나 변화했는지**를 구체적으로 설명하는 데에는 한계가 있다.

또한 직접 데이터를 대규모로 수집하여 범용 러닝 피로도 분류 모델을 학습하는 것은 3인 팀 방학 연구에서 현실적으로 어렵다. 따라서 본 연구는 공개 러닝·웨어러블 데이터셋을 활용하여 baseline 모델 또는 feature 설계 방향을 검토하고, 직접 수집한 소규모 러닝 데이터로 개인 기준선 기반 RFCI를 파일럿 검증하는 방식으로 진행한다.

## 4. 문제 정의

기존 러닝 앱은 다음과 같은 정보를 잘 제공한다.

| 제공 정보 | 예시 |
|---|---|
| 거리 | 5km |
| 시간 | 32분 |
| 평균 페이스 | 6분 24초/km |
| 심박수 | 평균 155bpm |
| 칼로리 | 350kcal |
| 운동 기록 | 날짜별 러닝 기록 |

그러나 본 연구에서 주목하는 문제는 다음과 같다.

> 사용자가 러닝 후반부에 피로해졌다고 느꼈을 때, 그 피로가 센서 데이터상에서 어떤 변화로 나타나는가?

추가로 다음 문제도 함께 다룬다.

> 공개 러닝·웨어러블 데이터 기반 분석 방법이 직접 수집한 개인 러닝 IMU 데이터에도 적용 가능한가?

## 5. 삼성 헬스와의 차이점

삼성 헬스와 같은 상용 앱이 이미 존재하기 때문에, 본 연구는 단순 운동 기록 앱을 만드는 것이 아니다.

| 구분 | 삼성 헬스 등 상용 앱 | 본 연구 |
|---|---|---|
| 목적 | 운동 기록 관리, 건강 관리, 코칭 제공 | 피로 변화 분석 방식의 실험과 검증 |
| 핵심 데이터 | 거리, 시간, 페이스, 심박수, 칼로리 등 | 공개 러닝 데이터, 스마트폰 IMU, 개인 기준선 |
| 분석 방식 | 완성된 상용 알고리즘 중심 | 공개 데이터 baseline + 개인 기준선 변화량 분석 |
| 결과 표현 | 기록 요약, 운동 점수, 훈련 계획 | 피로 변화 지수와 판단 근거 제공 |
| 차별점 | 사용자 서비스 완성도 | 연구 질문, 실험 비교, 설명 가능성 |
| 목표 | 운동 앱으로 사용 | 러닝 피로 변화 분석 가능성 검증 |

핵심 차이는 다음 문장으로 정리할 수 있다.

> 삼성 헬스는 운동 기록과 코칭을 제공하는 상용 서비스이고, 본 연구는 공개 러닝 데이터와 직접 수집 IMU 데이터를 활용하여 러닝 중 피로 변화가 어떤 센서 변화에서 발생했는지 설명하는 연구 시스템이다.

## 6. 기존 연구와의 차별점

기존 연구에서는 IMU 센서나 웨어러블 센서를 이용해 운동 상태를 분석하거나 러닝 피로도를 분류하는 연구가 이미 존재한다. 따라서 “IMU 센서로 피로도를 분류한다”는 것만으로는 차별점이 약하다.

본 연구는 다음 세 가지 방향으로 차별점을 만든다.

### 6.1 공개 데이터 기반 baseline과 직접 수집 데이터의 분리

공개 러닝·웨어러블 데이터셋은 baseline 모델 학습 또는 feature 설계에 활용한다. 반면 직접 수집 데이터는 모델 학습용 대규모 데이터가 아니라, 공개 데이터 기반 분석 방법과 개인 기준선 RFCI가 실제 개인 러닝 데이터에 적용 가능한지 확인하는 파일럿 테스트 데이터로 사용한다.

### 6.2 개인 기준선 기반 변화량 분석

사람마다 러닝 자세, 보폭, 흔들림, 페이스가 다르다. 따라서 본 연구는 러닝 초반 구간을 개인 기준선으로 설정하고, 이후 구간이 초반과 얼마나 달라졌는지를 분석한다.

예시:

| 항목 | 초반 구간 | 후반 구간 | 변화 |
|---|---:|---:|---:|
| 상하 흔들림 | 1.20 | 1.48 | +23.3% |
| 좌우 흔들림 | 0.85 | 1.02 | +20.0% |
| 페이스 | 6분 10초/km | 6분 55초/km | 저하 |
| 리듬 변동성 | 0.12 | 0.18 | +50.0% |

### 6.3 설명 가능한 피로 변화 지수

본 연구는 단순히 “피로도 높음”이라고 출력하지 않는다. 왜 피로도가 높게 나왔는지 판단 근거를 함께 제공한다.

```text
피로 변화 지수: 높음

판단 근거:
1. 후반 페이스가 초반 대비 12.1% 저하됨
2. 상하 흔들림이 18.4% 증가함
3. 좌우 흔들림이 9.8% 증가함
4. 리듬 불안정성이 15.6% 증가함
```

## 7. 연구 목적

본 연구의 목적은 다음과 같다.

1. AI Hub 또는 공개 러닝·웨어러블 데이터셋을 조사하여 러닝 피로 변화 분석에 활용 가능한 데이터 구조와 feature를 검토한다.
2. 공개 데이터 기반 baseline 모델 또는 feature 설계 방향을 정리한다.
3. Galaxy 스마트폰 IMU 센서와 러닝 로그를 이용해 직접 러닝 데이터를 소규모로 수집한다.
4. 직접 수집 데이터에서 러닝 초반 구간을 개인 기준선으로 설정하고, 이후 구간의 변화량을 분석한다.
5. 피로 변화 지수(RFCI, Running Fatigue Change Index)를 설계한다.
6. 공개 데이터 기반 분석 결과와 개인 기준선 RFCI 결과를 비교한다.
7. 피로 변화 판단 근거를 사용자가 이해할 수 있는 형태로 제공한다.

5. Galaxy Fit3와 Samsung Health의 심박수, 수면, 운동 기록은 러닝 피로 변화 해석에 보조 feature로 활용 가능한가?

## 9. 연구 가설

| 번호 | 가설 |
|---|---|
| H1 | 러닝 후반부에는 초반부에 비해 흔들림, 리듬 불안정성, 페이스 저하가 증가할 것이다. |
| H2 | 개인 기준선 대비 변화량 feature는 절대 센서값 feature보다 개인차를 줄이는 데 도움이 될 것이다. |
| H3 | 공개 러닝·웨어러블 데이터 기반 feature는 직접 수집한 러닝 IMU 데이터의 해석에도 일부 활용 가능할 것이다. |
| H4 | Galaxy Fit3와 Samsung Health의 심박수, 수면, 운동 기록은 RFCI 결과 해석에 보조 정보를 제공할 수 있을 것이다. |
| H5 | 피로 판단 근거를 함께 제공하면 단순 점수보다 사용자가 결과를 이해하기 쉬울 것이다. |

## 10. 데이터 활용 전략

본 연구는 데이터 활용을 세 단계로 나눈다.

| 구분 | 데이터 | 역할 |
|---|---|---|
| 공개 데이터 | AI Hub 또는 공개 러닝·웨어러블 데이터셋 | baseline 모델 학습 또는 feature 설계 |
| 직접 수집 데이터 | Galaxy 스마트폰 IMU 데이터 | 개인 기준선 RFCI 계산 및 외부 파일럿 테스트 |
| 보조 데이터 | Galaxy Fit3, Samsung Health | 심박수, 수면, 최근 운동 기록 보조 feature |

공개 데이터는 모델 학습 또는 feature 설계를 위한 기준 데이터로 사용하고, 직접 수집 데이터는 공개 데이터 기반 분석 방법이 실제 개인 러닝 데이터에 적용 가능한지 확인하는 외부 파일럿 테스트 데이터로 사용한다.

## 11. 공개 데이터셋 활용 계획

AI Hub 또는 공개 데이터셋을 검토할 때는 다음 우선순위를 적용한다.

| 우선순위 | 데이터 유형 | 활용 방식 |
|---|---|---|
| 1순위 | 러닝 + IMU + 피로도/강도 라벨 | baseline 모델 학습 후보 |
| 2순위 | 러닝 + IMU/웨어러블 센서 | feature 설계 및 모델 학습 후보 |
| 3순위 | 러닝 로그 + 심박수/페이스 | 보조 feature 설계 |
| 4순위 | 보행 + IMU | 러닝 데이터 부족 시 참고용 |
| 5순위 | 일반 피트니스/재활운동 | 원칙적으로 제외 또는 참고만 |

공개 데이터셋은 직접 수집 데이터의 대체재가 아니라, baseline 모델 학습 또는 feature 설계 참고용으로 활용한다.

## 12. 직접 데이터 수집 계획

### 12.1 수집 대상 데이터

본 연구의 직접 수집 데이터는 필수 데이터와 선택 데이터로 나누어 수집한다.

| 구분 | 데이터 | 설명 | 필수 여부 |
|---|---|---|---|
| 스마트폰 IMU | acc_x, acc_y, acc_z | 가속도계 데이터 | 필수 |
| 스마트폰 IMU | gyro_x, gyro_y, gyro_z | 자이로스코프 데이터 | 가능하면 필수 |
| 러닝 로그 | 거리 | 러닝 거리 | 필수 |
| 러닝 로그 | 시간 | 전체 러닝 시간 | 필수 |
| 러닝 로그 | 평균 페이스 | km당 평균 소요 시간 | 필수 |
| 러닝 로그 | 주관적 피로도 | 러닝 후 1~5점으로 기록 | 필수 |
| Galaxy Fit3/Samsung Health | 평균 심박수 | 운동 중 평균 심박수 | 선택 |
| Galaxy Fit3/Samsung Health | 최대 심박수 | 운동 중 최대 심박수 | 선택 |
| Galaxy Fit3/Samsung Health | 수면 시간 | 전날 수면 시간 | 선택 |
| Galaxy Fit3/Samsung Health | 케이던스 | 분당 걸음 수 또는 러닝 리듬 | 선택 |
| Galaxy Fit3/Samsung Health | 최근 운동 기록 | 최근 7일 운동량 | 선택 |

### 12.2 최소 수집 기준

본 연구는 대규모 모델 학습이 아니라 소규모 파일럿 검증을 목표로 한다.

| 항목 | 기준 |
|---|---|
| 참여자 수 | 3명 |
| 사람당 러닝 횟수 | 2~3회 |
| 총 러닝 세션 | 6~9회 |
| 러닝 시간 | 세션당 10~20분 |
| 주관적 피로도 | 러닝 직후 1~5점 기록 |

수집된 직접 데이터는 범용 피로도 모델 학습이 아니라 RFCI 계산 가능성 확인, 구간별 변화 시각화, 개인 기준선 방식 검토에 사용한다.

### 12.3 데이터 수집 방법

1. 스마트폰 센서 기록 앱을 사용하여 IMU 데이터를 CSV로 저장한다.
2. 러닝 후 거리, 시간, 평균 페이스를 기록한다.
3. Samsung Health에서 확인한 러닝 기록을 참고 데이터로 활용한다.
4. Galaxy Fit3에서 확인 가능한 심박수와 수면 기록은 선택 보조 데이터로 기록한다.
5. 주관적 피로도는 러닝 직후 1~5점으로 입력한다.

## 13. 데이터 구간 분할

러닝 데이터를 다음과 같이 나눈다.

| 구간 | 의미 |
|---|---|
| 초반 0~20% | 개인 기준선 구간 |
| 중반 20~70% | 일반 러닝 구간 |
| 후반 70~100% | 피로 변화 분석 구간 |

예를 들어 5km를 달렸다면 다음과 같이 나눌 수 있다.

| 구간 | 거리 |
|---|---|
| 초반 | 0~1km |
| 중반 | 1~3.5km |
| 후반 | 3.5~5km |

이때 초반 구간의 센서 특징을 개인 기준선으로 사용하고, 후반 구간이 초반과 얼마나 달라졌는지 계산한다.

## 15. 피로 변화 지수 RFCI 설계

본 연구에서는 피로 변화 지수를 RFCI라고 부른다.

**RFCI = Running Fatigue Change Index**

초기 버전의 RFCI는 다음 요소를 조합하여 계산한다.

| 구성 요소 | 의미 |
|---|---|
| 페이스 저하율 | 후반 페이스가 초반보다 얼마나 느려졌는가 |
| 상하 흔들림 증가율 | 후반부 상하 움직임이 얼마나 커졌는가 |
| 좌우 흔들림 증가율 | 후반부 좌우 움직임이 얼마나 커졌는가 |
| 리듬 불안정성 증가율 | 후반부 움직임 리듬이 얼마나 불규칙해졌는가 |

예시 계산 방식:

```text
RFCI = w1 * 페이스 저하율
     + w2 * 상하 흔들림 증가율
     + w3 * 좌우 흔들림 증가율
     + w4 * 리듬 불안정성 증가율
```

초기 연구에서는 모든 가중치를 동일하게 설정하고, 이후 공개 데이터 및 직접 수집 데이터의 결과를 보며 조정할 수 있다.

```text
w1 = w2 = w3 = w4 = 0.25
```

## 24. 윤리 및 안전 고려사항

1. 수집한 데이터는 연구 목적 외에는 사용하지 않는다.
2. 개인 식별 정보는 저장하지 않는다.

4. 운동 중 무리한 실험을 강요하지 않는다.
5. 피로 분석 결과를 의학적 진단으로 표현하지 않는다.
6. 부상 위험, 질병 예측, 치료 조언과 같은 표현은 사용하지 않는다.
7. AI Hub 또는 공개 데이터셋은 라이선스와 이용 조건을 확인한 후 사용한다.

5. 후반부 움직임이 초반과 얼마나 달라졌는지 분석한다.
6. 피로 변화 지수 RFCI를 설계한다.
7. 피로 판단 근거를 사용자에게 설명한다.
